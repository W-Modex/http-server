Security notes

This page summarizes practical threat coverage in the current implementation.

Threats considered

  • Cross-site request forgery on state-changing endpoints.
  • Session fixation and session token abuse risks.
  • OAuth callback replay or forged state values.
  • Path traversal attempts against static file serving.

Protections implemented

Cookie hardening

Session cookies use hardening flags including HttpOnly, Secure, and SameSite policies.

CSRF tokens

State-changing requests are checked against per-session CSRF token values.

OAuth state validation

Authorization flow state is stored and validated before code exchange completes.

Safe static paths

Path normalization rejects traversal patterns and constrains file lookup behavior.

Known limitations

  • This is an educational/demo server, not a completed production hardening profile.
  • Development certificates are self-signed for local HTTPS testing.
  • No formal third-party security audit has been performed.
  • Additional controls like rate limiting and deeper observability are still planned.

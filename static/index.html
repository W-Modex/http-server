Low-level HTTPS Web Server in C (from scratch)

HTTP/1.1 parsing • poll() event loop + worker pool • TLS (OpenSSL) • sessions + CSRF • Google OAuth2 • PostgreSQL + migrations • Dockerized

View Architecture Run with Docker GitHub Repo

Why this exists

  • Understand web backends under the hood without framework abstractions.
  • Implement security primitives directly: sessions/cookies, CSRF, and OAuth2 state handling.
  • Ship a portable demo with Docker Compose, PostgreSQL, and automatic migrations.

Feature overview

HTTP/1.1 parser

Request line, headers, query/body extraction, and typed method handling.

Routing (method + path)

Explicit route table dispatching handlers by HTTP method and URL path.

poll() event loop

Non-blocking readiness loop for accept/read/write connection handling.

Worker thread pool

Queued request jobs processed concurrently by pthread worker threads.

TLS termination (OpenSSL)

TLS handshake, certificate loading, and encrypted transport via OpenSSL.

Static files + MIME

Path resolution, safe file reads, and content type selection for assets.

Sessions (secure cookies)

Server-side session store with cookie flags and session lifecycle controls.

CSRF protection

Per-session token generation and verification for state-changing requests.

Google OAuth2

Authorization code flow with state management and callback processing.

PostgreSQL + migrations

Persistent auth/session data with SQL migrations applied in Docker startup.

Quick demo

git clone https://github.com/W-Modex/http-server
cd http-server
docker compose up --build

Open https://localhost:3434 and accept the self-signed certificate warning.
